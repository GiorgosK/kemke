A. Connecting

 

You need user:pass to accesss the API  

curl  -u user:pass
  -X POST https://kemke.webx2.com/api/incoming \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{}'

 

If not correct user:pass you will get back 

{"error":"Access denied."}

 

For tests one can use api_user:api_user 

~ ❯❯❯ curl  -u api_user:api_user \
  -X POST https://kemke.webx2.com/api/incoming \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{}'

 

which warns that document subject and document sender properties are required: 

{
    "error": "Request to https:\/\/kemke.ddev.site\/api\/incoming failed with status 422: Validation failed. Details: [\"Document entry 0 requires a sender value.\",\"Document entry 0 requires a subject value.\"]"
}

C. To upload large files in chunks 

 

use /api/incoming/files 

 

1) Start a session and get an upload_id 

curl -u api_user:api_user \
  -X POST https://kemke.webx2.com/api/incoming/files/init \
  -H 'Content-Type: application/json' \
  -d '{
    "filename": "big.pdf",
    "mime_type": "application/pdf",
    "expected_chunks": 2,
    "total_size": 12,
    "chunk_size": 6
  }'

returns something like 

{"upload_id":"50dd...c3","filename":"big.pdf","mime_type":"application/pdf","expected_chunks":2,"chunk_size":6,"total_size":12}

 

2) Send chunks in order (chunk_index starts at 0) 

curl -u api_user:api_user \
  -X POST https://kemke.webx2.com/api/incoming/files/chunk \
  -H 'Content-Type: application/json' \
  -d '{
    "upload_id": "50dd...c3",
    "chunk_index": 0,
    "data": "Zmlyc3QtY2h1bms="   // base64 for the first chunk
  }'

which replies with how many chunks/bytes it has so far: 

{"upload_id":"50dd...c3","received_chunks":1,"received_size":11}

Repeat with chunk_index 1, 2, ... until all chunks are sent.

 

3) Finalize to create the managed file 

curl -u api_user:api_user \
  -X POST https://kemke.webx2.com/api/incoming/files/complete \
  -H 'Content-Type: application/json' \
  -d '{ "upload_id": "50dd...c3" }'

success returns the file details you can attach to an incoming: 

{"upload_id":"50dd...c3","fid":321,"uuid":"0e4...a1","filename":"big.pdf","mime_type":"application/pdf","size":12,"uri":"public://documents/big_1.pdf","url":"https://kemke.webx2.com/sites/default/files/documents/big_1.pdf"}

 

(Optional) Cancel a session 

curl -u api_user:api_user \
  -X POST https://kemke.webx2.com/api/incoming/files/cancel \
  -H 'Content-Type: application/json' \
  -d '{ "upload_id": "50dd...c3" }'

 

4. To finally create the incoming we use the fid  the uploaded file in /api/incoming 

curl -u api_user:api_user \
  -X POST https://kemke.webx2.com/api/incoming \
  -H 'Content-Type: application/json' \
  -d '{
    "documents": [
      {
        "sender": "Ονομα Επώνυμο 10",  
        "subject": "θέμα εισερχομένου 10",  
        "files": [
          { "fid": "321" }
        ]
      }
    ]
  }'
